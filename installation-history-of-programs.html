Installation history of programs

editor

Sublime Text

  • why getbootstrap.com recommended me to use it when I was reading the part of Getting started
  • Installation: manual
  • The installation package location:"~/install_package/tar/Sublime Text 2.0.2.tar.bz2"
  • The executive file location:"~/install_package/tar/Sublime Text 2/"

autojump

  • why help you change directory more quickly,and I know this tool from the blog of Jenny
  • Installation: apt-get install autojump

gph-import

  • why docs.getpelican.com says it makes it really easy to push the content of the output dir generated by Pelican to a repository’s gh-pages branch on GitHub. Howerver,I couldn't understand
  • Installation: pip install ghp-import

Pelican

  • why a static site generator,written in python
  • Installation: pip install pelican
  • date:2013-11-08

mczip

  • why 带中文字符文件名的zip压缩包打开用unzip解压后，文件名和目录名都是乱码，gooogle一下，很多给出的是用unzip -O CP639这样的命令，但是现在unzip 好像没有了-O这个参数。还有很多是推荐用convmv 转换编码格式，不知道为什么不管用。然后在这个博客找到了这个小软件mczip
  • Installation: /usr/local/bin/mczip
  • date: 2013-11-30

texlive

  • why 起因是翻译矩阵论文，里面有很多公式，在linux环境下怎么高效得去编辑含有公式的文档呢，于是想到了tex，做毕业设计的时候大东就是用它写的论文。
  • Installation: /usr/home/smallest/texlive/2013
  • date: 2013-12-05

jhead

  • why 需要旋转图像的方位来保证其在浏览器中的正确显示.jhead可以修改文件的Exif(Exchangeable Image File,is a kind of JPEG)头部信息。
  • Installation:apt-get install jhead
  • date: 2013-12-14

libpdf(chrome)

  • why 用chromium看pdf文档，但是chromium不带这项功能，从chrome的安装包中提取libpdf放到/usr/lib/chromium
  • Installation:ar x google-chrome-stable;tar --lzma -xvf data.tar.lzma;cp opt/google/chrome/libpdf.so /usr/lib/chromium;
  • date: 2013-12-14
